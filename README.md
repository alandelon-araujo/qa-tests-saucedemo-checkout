# 🧪 Testes QA – Fluxo de Checkout (SauceDemo)
🔗 Sistema testado: https://www.saucedemo.com/

Este projeto apresenta a execução completa de testes manuais no fluxo de checkout da aplicação SauceDemo, simulando um cenário real de validação de sistema.

---

## 🎯 Objetivo

Validar o fluxo de compra do usuário, desde o login até a finalização do pedido, garantindo o correto funcionamento das funcionalidades.

---

## 🔍 Escopo dos Testes

* Login de usuário
* Seleção de produtos
* Carrinho de compras
* Preenchimento de dados no checkout
* Finalização da compra

---

## 🧪 Tipos de Testes Realizados

* Testes Funcionais
* Testes Negativos
* Testes Exploratórios
* Validação de dados
* Testes de usabilidade básica

---

## 📄 Artefatos Produzidos

* Plano de Testes
* Casos de Teste
* Bug Reports com evidências
* Relatório de execução

---

## 🐞 Bugs Encontrados (Destaque)

* Checkout permite finalizar sem produtos no carrinho
* Falha de validação nos campos do formulário
* Botão na interface não responde ao clique

---

## 🧠 Análise Crítica

Durante os testes, foram identificadas falhas relevantes no fluxo de checkout, incluindo problemas de validação de dados e inconsistências no comportamento do sistema.

Essas falhas impactam diretamente a experiência do usuário e a confiabilidade da aplicação, podendo resultar em pedidos inválidos e perda de credibilidade.

---

## 🛠️ Ferramentas Utilizadas

* Testes manuais
* Qase.io (gestão de testes e bugs)
* Navegador Web

---

## 📊 Resultado

Foram executados testes end-to-end cobrindo fluxos principais, alternativos e de exceção, com identificação de bugs críticos e análise do comportamento do sistema.

---

📌 Projeto com foco em simular atividades reais de um Analista de QA.
